20 // Header for resolving SVC call for function which we don't want to implement.
21 // This exists because of many Symbian devs make their own allocator and switch it
22 // Some functions are overriden with HLE functions instead of just implementing the svc call
23 // Because of its complex structure (NMutex, NThread) change over different Symbian update
24 // But, the public structure still the same for them (RThread, RFastLock), contains only
25 // a handle. This give us advantage.
