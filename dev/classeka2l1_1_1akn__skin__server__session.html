Cancel a pending signal request, if there is one available.

It will be cancel only if 2 conditions are sastified:

  • No pending notifications.
  • A notify handler must be set.

◆ do_next_event()

void eka2l1::akn_skin_server_session::do_next_event ( service::ipc_context *  ctx)
private

Get the next event in the notify list and notify the waiter.

  • If cancel flag is set, the client will be notify with code KErrCancel.
  • If the notify list is just empty, the requester of this next event will be the next one waiting.
  • Else, the first element in the queue is popped and the client will be notify will that code.
