91  // Samsung Galaxy S7 devices (Exynos 8890) have a big.LITTLE configuration where the cacheline size differs between big and LITTLE.
92  // GCC's cache clearing function would detect the cacheline size on one and keep it for later. When clearing
93  // with the wrong cacheline size on the other, that's an issue. In case we want to do something different in this
94  // situation in the future, let's keep this as a quirk, but our current code won't detect it reliably
95  // if it happens on new archs. We now use better clearing code on ARM64 that doesn't have this issue.
